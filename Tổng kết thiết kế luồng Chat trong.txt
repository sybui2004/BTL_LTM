II. Các loại Chat và đặc điểm
Private Chat

Chat 1-1 giữa users
Lưu trữ trong DB
Hiển thị status online/offline
Không phụ thuộc game state
World Chat

Chat toàn server
Giữ 100 tin nhắn gần nhất
Không lưu DB
Broadcast tới tất cả users online
Match Chat

Chat trong trận đấu
Temporary (xóa sau khi trận kết thúc)
UI tích hợp với match screen
Phụ thuộc match state
Lobby Chat

Chat trong phòng chờ
Temporary (xóa khi room giải tán)
UI tích hợp với lobby screen
Phụ thuộc room state
III. Cơ chế tích hợp
ChatContext Interface:
public interface ChatContext {
    String getChannelId();      // Định danh kênh chat
    ChatType getType();         // Loại chat
    UserSummary getCurrentUser();
    List<UserSummary> getParticipants();
    boolean canSendMessage();   // Kiểm tra điều kiện chat
}
ChatComponent:
Component có thể reuse
Nhận ChatContext để adapt với từng môi trường
Tự handle UI updates và message sending
Có thể embed vào các màn hình khác
IV. Luồng xử lý
Message Flow
User Input -> ChatComponent -> ChatService -> TCP/WebSocket -> BE
     ^          |
     |          v
UI Update <- ChatContext (validate state)
Integration Flow:
MatchScreen/LobbyScreen
     |
     v
ChatContext (implements interface)
     |
     v
ChatComponent (reusable UI + logic)
     |
     v
ChatService (communication with BE)

V. Lợi ích thiết kế
Tính module hóa

Các components độc lập
Dễ maintain và test
Có thể phát triển song song
Tái sử dụng code

Common UI components
Shared message handling
Base implementations
Dễ mở rộng

Thêm loại chat mới dễ dàng
Không ảnh hưởng code hiện tại
Interface rõ ràng
Phù hợp team work

Ít conflicts khi merge
Clear boundaries giữa các phần
Dễ phân chia công việc
VI. Points to Consider
Dependencies

Match system cần expose match state
Lobby system cần expose room state
Cần định nghĩa các interfaces trước
State Management

Cần handle các edge cases (disconnect, timeout)
Cleanup resources khi switch screens
Sync state giữa các clients
Performance

Message caching strategy
UI virtualization cho large message lists
Batch updates để tránh UI freezing
Security

Validate messages server-side
Rate limiting
Access control based on context
Đây là blueprint tổng thể cho việc implement chat system. Team có thể bắt đầu với base implementations và progressively enhance từng phần theo needs.
